# `data/` — pack telemetry, weather, and derived files

This folder is the **persistent state of the cabin install**. The
autonomous loop appends to most files every iteration and pushes
the snapshot to GitHub, so re-cloning the repo gives you the full
history.

## Source-of-truth files (logger-written, append-only)

| File | What it is |
|------|------------|
| **`pack.csv`** | Every BLE poll of both batteries. ~ one row per 10 s. Columns: `ts, state, pack_v, pack_i, pack_p, soc_a, soc_b, v_a, v_b, i_a, i_b, temp_a, temp_b, rem_a, rem_b, delta_v_a, delta_v_b, smoothed_i, smoothed_p, minutes_remaining, name_a, name_b`. Written by `scripts/log.py`. Grows ~ 13 MB / day; this is the canonical source of truth. |
| **`weather.csv`** | Open-Meteo observation every 30 min. Columns: `ts, lat, lon, temperature_c, cloud_cover_pct, shortwave_radiation_wm2, wind_speed_ms, wind_gusts_ms, weather_code, is_day, sunrise_iso, sunset_iso, shortwave_radiation_sum_today_wh_m2, uv_index_max_today`. Written by `scripts/weather.py`. |
| **`pack.log`** | Human-readable progress / BLE-flap notes from `scripts/log.py`. Useful when debugging connection issues. |
| **`launch.log`** | The `Volthium Monitor.app` launcher's chatty log of when services started / failed. |
| **`weather.log`** | Reserved for `scripts/weather.py` output (currently mostly empty). |

## Derived / rolled-up files (loop-regenerated)

| File | What it is |
|------|------------|
| **`daily_summary.csv`** | One row per calendar date with day-total stats (charge/discharge Ah, generator minutes, weather kWh/m², partial flag, etc.). Written by `scripts/daily_summary.py`. Input to `volthium.solar_model.SolarModel.fit_from_daily_summary`. |
| **`calibration_log.csv`** | History of meaningful `SolarModel` coefficient changes. Each row: `ts, coefficient, n_observations, confidence, source, notes`. Written by `scripts/calibration_log.py` (called from `generator_advisor.py`). |
| **`projection_log.csv`** | History of advisor projection snapshots. Each row: `ts, start_soc_pct, projected_sunrise_soc, projected_tomorrow_evening_soc, projected_low_soc, solar_model_coefficient, today_irradiance_kwh_m2, sunrise_iso, source`. Rate-limited to ≥ 25 min apart. Foundation for `scripts/projection_accuracy.py`. |
| **`voltage_soc_table.csv`** | OCV (open-circuit voltage) → SOC% lookup table extracted from rest-window samples. Written by `scripts/voltage_soc_calibration.py`. Designed to be re-run periodically as the table sharpens with more data. |

## Reports

| Path | What it is |
|------|------------|
| **`reports/YYYY-MM-DD.md`** | Markdown day-report generated by `scripts/end_of_day_report.py`. Sections: Pack (SOC walk + peaks), Solar harvest, Weather, SolarModel state (calibration_log entries dated to the day), Projection accuracy (validated projections), Cross-references. Idempotent — re-runs overwrite with the freshest snapshot for the current day; historical days stay frozen. |

## Archived backups

| File | What it is |
|------|------------|
| **`pack.csv.v0-1512`** | Snapshot of the pack.csv taken on 2026-05-17 at 15:12 before the schema changed (added `name_a` / `name_b` columns). Preserves the original charge → discharge transition data point. |

## Sizing notes

`pack.csv` grows ~ 13 MB / day at the current 10-s polling cadence
and is the main contributor to repo size growth. Over a month
that's ~ 400 MB. If the repo gets unwieldy we'll revisit log
rotation (e.g. archive older days to `pack-YYYY-MM-DD.csv` files
and keep only the last week unrolled).

## Dashboard surfaces

The live dashboard renders most of this data into a single page;
the log-style files also have their own browseable pages:

- `/calibration` — `calibration_log.csv` as a table
- `/projections` — `projection_log.csv` as a table
- `/accuracy` — projection-vs-actual diff (uses `projection_log.csv` + `pack.csv`)
- `/reports` — index of `reports/*.md`
- `/today-report` — live regeneration of today's day-report
- `/report/YYYY-MM-DD` — specific historical day-report
